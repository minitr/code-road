# 集合框架

# Iterator接口

迭代器接口

hasNext()：判断是否存在下一个

next()：返回集合种下一个对象

remove()：删除调用next返回的对象

# Iterable接口

内部提供了Iterator接口，并且提供了forEach，这是一个语法糖，本质上还是调用迭代器遍历

ListIterator接口继承Iterator接口，在遍历List集合时可以从任意索引下标开始遍历，而且支持双向遍历

# Map和Collection接口

# Map接口

HashMap

底层使用数组加链表+红黑树三种结构实现

发生哈希冲突时，会将相同映射的元素连成一条链表，如果长度大于8，且数组长度大于64，则会转换为红黑树

LinkedHashMap

在HashMap的基础上添加了一条双向链表

TreeMap

默认情况下按照key自然排序，或者传入定制的Comparator进行自定义规则排序，也是按照key值！

如果要按照value值要转为entry的list再排序

    public static void main(String[] args) {
        TreeMap<Integer, Integer> treeMap = new TreeMap<>();
        treeMap.put(1, 3);
        treeMap.put(2, 4);
        List<java.util.Map.Entry<Integer, Integer>> entrySet = new ArrayList<>(treeMap.entrySet());
        entrySet.sort((o1, o2) -> {
            return Integer.compare(o2.getValue(), o1.getValue());
        });
        for (java.util.Map.Entry<Integer, Integer> s : entrySet) {
            System.out.println(s.getKey() + " " + s.getValue());
        }
    }

自然排序：

​ key必须实现Comparable接口，例如Integer类，默认从小到大

定制排序：

​ 初始化TreeMap时传入比较器，可用lambda表达式

# Collection接口

# Set

所有的Set都应该要有相同的hashCode和equals方法

SortedSet

扩展了排序行为

HashSet

底层通过HashMap实现，创建一个HashMap，使用一个静态常量PRESENT作为Map的value值，只需要关注插入的key，屏蔽value即可，存入Set的元素状态最好不要发生变化

LinkedHashSet

通过LinkedHashMap实现，再底层才是HashMap

TreeSet

底层采用TreeMap实现，判断元素是否重复的方法是compareTo方法是否返回0，而不是hashcode和equals方法了

# List

ArrayList

底层用数组作为存储结构，查找快，增删慢

首次扩容后长度为10，调用add需要计算容器最小容量，第二次开始扩容为1.5倍

LinkedList

采用双向链表存储数据，增删快，查询慢

没有扩容机制，底层实现了Deque接口，可以作为双端队列

# Quque

提供了两套增加、删除的API

  1. add，remove，get，操作失败抛出异常
  2. offer，poll，peek 操作失败返回失败默认值

ArrayDeque

使用数组实现的双端队列，作为栈时比Stack好，作为队列时比LinkedList好

PriorityQuque

排序队列，与之前的方式一样，要么元素实现了Comparable接口，要么传入定制比较器进行排序

# Stream流操作

Stream表示元素序列，并支持对这些元素进行不同类型的计算操作，Stream 操作包括中间操作和终端操作。中间操作返回Stream，这样我们就可以在不使用分号的情况下串联多个中间操作，终端操作返回void或者一个非Stream结果值。

例如filter, map 和 sorted 是中间操作，而forEach是一个终端操作。

创建一个简单的用户类

public class User {
    private String name;
    /**
     * 0代表男性 1代表女性
     */
    private Integer sex;
    private Integer age;
}

创建列表和流

List<User> userList=new ArrayList<>();
userList.add(new User("user1",0,25));
userList.add(new User("user2",0,18));
userList.add(new User("user3",1,30));
userList.add(new User("user4",0,41));
userList.add(new User("user5",1,27));
Stream<User> stream = userList.stream();

# 分组

在数据库中经常实现分组查询，而为了实现在内存中对list进行分组，可以使用stream流

根据性别分组

stream.collect(Collectors.groupingBy(user -> user.getSex()));

返回值

{0=[User{name='user1', sex=0, age=25}, User{name='user2', sex=0, age=18}, User{name='user4', sex=0, age=41}], 
1=[User{name='user3', sex=1, age=30}, User{name='user5', sex=1, age=27}]}

# 排序

根据年龄排序，并取前几个

顺序

stream.sorted(Comparator.comparing(User::getAge)).limit(3).collect(Collectors.toList());

[User{name='user2', sex=0, age=18}, User{name='user1', sex=0, age=25}, User{name='user5', sex=1, age=27}]

逆序

stream.sorted(Comparator.comparing(User::getAge).reversed()).limit(3).collect(Collectors.toList());

[User{name='user4', sex=0, age=41}, User{name='user3', sex=1, age=30}, User{name='user5', sex=1, age=27}]

# where

最值筛选

获取最大/最小的对象

// 最小
Optional<User> min = userList.stream()
	.min(Comparator.comparing(User::getAge));
// 最大
Optional<User> max = userList.stream()
	.max(Comparator.comparing(User::getAge));

// 获得对象
User user = min.get();

条件筛选

通过filter过滤器和lambda表达式进行筛选

年龄大于30

stream.filter(user -> user.getAge() > 30).collect(Collectors.toList());

# 去重

获取年龄小于30岁的用户的名字，并且去重

stream.filter(user -> user.getAge() < 30)
    .map(User::getName)
    	.distinct()
    		.collect(Collectors.toList());

# apply

给属性批量赋值

stream.forEach(user -> user.setAge(30));

对某个字段进行处理

userList.stream()
	.map(user -> {user.setName(user.getName().replaceAll("\u0000", "")); return user;})
	.collect(Collectors.toList());

# 提取属性

提取多个属性组成map

stream.collect(Collectors.toMap(User::getName, User::getSex));

# 求和

对年龄求和

stream.mapToInt(User::getAge).sum();

# Reduce

reduce操作将stream的所有元素合并到一个结果中。

获取年龄最大的用户

stream.reduce((user1,user2)-> user1.getAge()>user2.getAge()?user1:user2)
    .ifPresent(System.out::println);
